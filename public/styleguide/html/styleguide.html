Accessibility

Accessibile Use of Color

Color combinations must meet at least WCAG 2.0 level AA contrast ratios.
Level AA: at least 4.5:1 for normal text and 3:1 for large text (18px & BOLD or 24px).
Level AAA: at least 7:1 for normal text and 4.5:1 for large text.

Primary Buttons

Buttons

Buttons are used for actions a user can take on a page. Their visual design is intended to make them stand out against content to make it more apparent what actions are available to a user.

We do not disable buttons. Disabled buttons present accessibility issues and can confuse users as to how to enable them. So instead, we allow users to press any button and show a message if something must be done before that action can be taken.

Primary Buttons

Default buttons are to be used in most situations that require a user to take an action.

State: Default
State: Hover
State: Pressed

Inverted Buttons

Inverted Buttons

To be used for secondary actions or demphasizing an option (ex: a ‘Go Back’ button next to a default button for ‘Continue’).

State: Default
State: Hover
State: Pressed

Accent Buttons

Accent Buttons

Accent buttons are to be used where there are multiple buttons on the screen and we want to call attention to the most important action on the screen. Only one accent button should be visible on the screen at a time.

State: Default
State: Hover
State: Pressed

TextIcon Buttons

Text + Icon Buttons

Text+Icon buttons are to be used when tied to an action that both has an associated icon and the interface uses only the icon to refer to this action elsewhere in the design.

State: Default
State: Hover
State: Pressed

Icon Buttons

Icon Buttons

Icon buttons are to be used only when the action and icon are common enough interactions that no text is necessary (ex: ‘Previous’ and ‘Next’ with the chevron icon). Even then, text may still be associated with the button.

Text+Icon buttons are to be used when tied to an action that both has an associated icon and the interface uses only the icon to refer to this action elsewhere in the design.

State: Default
State: Hover

Link Buttons

Link Buttons

Text Only Buttons function similarly to links, but do not have a visited state. Text Only Buttons should always be placed next to a standard button and are used to deemphasize an option even more than the inverted button, such as ‘Cancel’ next to an ‘Apply’ button.

State: Default
State: Hover
